Getting Started with Scaffold-ETH 2

Scaffold-ETH 2 is a powerful, open-source toolkit for building decentralized applications (dApps) on Ethereum and EVM-compatible blockchains. It provides a batteries-included setup with a local development chain, smart contract hot-reloading, a frontend built with Next.js, and various helpful hooks and components to interact with your smart contracts.

This guide will walk you through creating a new Scaffold-ETH 2 project, running it locally, and making basic modifications to the example smart contract.

1. Create Your Scaffold-ETH Project​

To start, use the create-eth CLI tool, which will set up a new Scaffold-ETH 2 project for you.

Open your terminal and run:

npx create-eth@latest

Follow the prompts, selecting the default options if unsure.

Once the project is created, navigate into the project directory and install the dependencies:

cd my-dapp-example
yarn install

2. Run the Local Development Environment​

Scaffold-ETH 2 requires three separate terminal instances for a complete local development experience:

Terminal 1: Start the Local Blockchain​

This command starts a local Hardhat blockchain node, providing a simulated Ethereum environment for development and testing.

yarn chain

You'll see a list of available accounts and their private keys, which you can use in your development wallet (e.g., MetaMask).

Terminal 2: Deploy Your Smart Contract​

This command compiles and deploys your smart contracts (initially, the YourContract.sol example) to the local Hardhat network started in the previous step.

yarn deploy

Every time you make a change to your contracts in packages/hardhat/contracts/, you'll need to re-run this command. For convenience, you can use yarn deploy --watch to automatically redeploy on changes.

Terminal 3: Start the Frontend Application​

This command starts the Next.js frontend development server.

yarn start

Your application will typically be available at http://localhost:3000.

3. Understanding the Project Structure​

A Scaffold-ETH 2 project is a monorepo managed with Yarn workspaces. The main packages are:

  • packages/hardhat: Contains your smart contract code, deployment scripts, and Hardhat configuration.
    • contracts/: Your Solidity smart contracts (e.g., YourContract.sol).
    • deploy/: Scripts used by yarn deploy to deploy your contracts.
  • packages/nextjs: Contains your frontend Next.js application.
    • app/: The main application code following Next.js App Router conventions.
    • components/: Reusable React components.
    • hooks/scaffold-eth/: Useful React hooks for interacting with contracts, wallet, etc.

4. Modifying the Smart Contract​

Let's add a new function to the example contract.

a. Add getGreeting Function​

Open my-dapp-example/packages/hardhat/contracts/YourContract.sol and add the following getGreeting function:

packages/hardhat/contracts/YourContract.sol
contract YourContract {
    // ... existing state variables and functions ...

    /**
     * @notice Returns the current greeting.
     * @return greeting_ The current greeting.
     */
    function getGreeting() public view returns (string memory greeting_) {
        greeting_ = greeting;
    }

    // ... rest of the existing code ...
}

Remember to re-deploy your contract after saving changes: Go to Terminal 2 and run yarn deploy.

b. Viewing the New Function in the Debug UI​

Scaffold-ETH provides a debug UI (usually at http://localhost:3000/debug) where you can interact with your deployed contracts. To ensure that parameter-less view functions like getGreeting appear in the debug UI, you might need to adjust the filtering logic in the ContractReadMethods.tsx component.

Open my-dapp-example/packages/nextjs/app/debug/_components/contract/ContractReadMethods.tsx. Modify the functionsToDisplay filter as follows:

packages/nextjs/app/debug/_components/contract/ContractReadMethods.tsx
const functionsToDisplay = (
  ((deployedContractData.abi || []) as Abi).filter(
    (part) => part.type === "function"
  ) as AbiFunction[]
)
  .filter((fn) => {
    const isQueryable =
      fn.stateMutability === "view" || fn.stateMutability === "pure";
    return isQueryable;
  })
  .map((fn) => {
    return {
      fn,
      inheritedFrom: (
        (deployedContractData as GenericContract)
          ?.inheritedFunctions as InheritedFunctions
      )?.[fn.name],
    };
  })
  .sort((a, b) =>
    b.inheritedFrom ? b.inheritedFrom.localeCompare(a.inheritedFrom) : 1
  );

After this change and restarting your frontend (if it didn't hot-reload), the getGreeting function should now appear under the read methods in the debug UI.

5. Next Steps & Further Learning​

You now have a basic Scaffold-ETH 2 project running! Here are some things you can explore next:

  • Build out your contract: Add more state variables and functions to YourContract.sol (or rename it).
  • Create custom UI components: In packages/nextjs/components/, build React components to interact with your contract using the provided hooks (e.g., useScaffoldContractRead, useScaffoldContractWrite).
  • Explore Scaffold-ETH hooks: Check out the hooks in packages/nextjs/hooks/scaffold-eth/ for wallet interactions, network status, and more.
  • Learn about deploying to testnets/mainnet: The yarn deploy command can be configured for other networks.

For more in-depth information, refer to the official documentation:

  • Scaffold-ETH 2 Documentation: https://docs.scaffoldeth.io/
  • Hardhat Documentation: https://hardhat.org/
  • Next.js Documentation: https://nextjs.org/docs

Happy building!
